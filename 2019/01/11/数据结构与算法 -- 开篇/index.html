数据结构与算法 -- 开篇

发表于 2019-01-11 | 阅读次数:
字数统计: 2.4k | 阅读时长 ≈ 7

程序猿都知道，数据结构与算法是基础，大学的时候是必修课。但即使是最基础的东西，也有很多人coding多年，对此还是稀里糊涂一知半解，所以也很难应用在工作中。更有甚者从来都没考虑过，算法、性能这些东西，写的代码只要功能可以实现就可以了，其它根本就不会考虑。这种程序员应该不在少数吧？但如果你想成为技术大牛，“真正”的学会编程，想要跳槽去BAT，数据结构是你必须熟练掌握的技能。

整理这个系列，是因为有几个刚刚学习编程的朋友，经常问我一些相关内容。而且本人刚刚“人肉翻墙”，正在找工作ing，外国公司比较注重算法方面的知识，面试笔试经常考这些问题，所以正好可以巩固一下相关知识。写博客的目的本来就是整理沉淀自己，因此将数据结构与算法作为一个技术博客的开篇，也是极好的。

此系列仅是一个学习笔记，内容来自于极客时间—数据结构与算法之美(讲师：王争)和极客时间—算法面试通关40讲(讲师：覃超)，我会将两个课程的主要知识点结合自己的一些理解、想法和观点总结到一起，记录下来方便学习和回顾。内容仅是一些常用的算法和技巧以及在实际中的应用，并不是包含数据结构课程中的全部内容。两位讲师讲解的还是不错的，想好好学习的童鞋可以点击上面链接订阅学习。

为什么要学习数据结构与算法

之前已经说过，很多人写了很多年代码，即使不懂数据结构与算法，也没有影响工作，功能依然可以实现，产品可以照常发布。所以不会也无所谓，是这样吗？

  • 面试

    很多大公司，比如BAT等大厂和一些外企公司，面试的时候都喜欢考算法、让人现场写代码。有些人虽然技术不错，但每次去面试都会“跪”在算法上。

    尤其是一些国外的互联网公司，更是注重当场写算法，他们通常会在电话面试的时候，就要求写算法了。面试官提出题目，让面试者在线写出程序算法，常用的网站有collabedit和coderpad等。他们通常不在乎你用什么语言，会什么样的技术，连这些基础算法都搞不明白，那些所谓的技术真的可以用的融会贯通吗？

    想获得一些国外面试方面的经验，可以到一亩三分地上去看一看。

  • 修炼内功

    看过武侠小说的都知道，内功心法是学习武功的基础。张无忌学习了《九阳真经》之后，内功深厚，学习其它武功招式也是信手拈来，分分钟就学会了《乾坤大挪移》这种绝世武功。而数据结构与算法就是程序员的内功心法，学通之后，再学习各新的语言、新的技术，也会快很多。

    比如现在很火的技术区块链，就和单向链表如出一辙。而且基于基本的数据结构，现在的区块链已经有用树等结构实现的技术出现。如果明白数据结构，可以更有利于的理解学习区块链技术。

    再比如说，现在是一个大数据时代，动辄几十亿条数据，要处理这些数据怎么办？像平时编程一样，直接排序、查找、遍历？且先不说效率问题，就是这些数据要占多少内存恐怕也吃不消吧。

    但是同样一个功能，为什么别人写的运行速度很快，资源占用很低，扩展性很好，还可以拿到开源社区给很多人用。

    所以，如果你也想像大牛一样，很快学会新的技术，写出拿的出手的code，数据结构与算法是必须要掌握的基础知识。

如何有效学习数据结构与算法

很多人都知道数据结构与算法的重要性，也想学好它们，但学习起来真的是非常枯燥、抽象难懂、看不下去，学起来又放下去，再学起来又放下去，如此反复，最终也没有成功。

但真正的原因是没有找到好的学习方法，没有抓住学习的重点。实际上，常用的、基础的知识点的东西并不多。只要掌握了正确的学习方法，学起来并没有看上去那么难。算法不需要死记硬背，而是要多理解、多训练。

  • Chunk it up （切碎知识点）

    学习一个新的内容，最好的方式是先将其分解成一个个知识点，然后注重每个知识点之间的脉络联系，将其串成一个知识体系，循序渐进。只有将所有的知识点变成了一个体系，找到一条主干，然后将知识点挂在叶子上，才会记忆的更加牢固，理解的更加透彻。

    这是一张主要数据结构和算法的结构图。作为初学者，或者一个非算法工程师，并不需要掌握图里面的所有知识点。很多高级的数据结构与算法，比如二分图、最大流等，这些在平时开发中很少会用到，就可以暂时不用看。总结一下比较常用需要重点关注的知识点，掌握了这些基础，再学更加复杂的数据结构和算法，就会非常容易、非常快：

    数据结构：数组、链表、栈、队列、散列表、二叉树、堆、跳表、图、Trie 树等；

    算法：递归、排序、二分查找、搜索、哈希算法、贪心算法、分治算法、回溯算法、动态规划、字符串匹配算法等。

  • Deliberate practicing （刻意训练）

    • 刻意练习

      要将每个知识点单独练习、重复练习、反复练习，增强自己的理解记忆。

    • 练习缺陷、弱点地方

      尤其要注重自己不熟悉、有缺陷的地方，更要多多练习。

    • 不舒服、不爽、枯燥

      在练习的时候，会觉着很不舒服，很枯燥，反而会出现水平下滑的现象。没关系，坚持住，这是因为它正在打破一些已经固有的思维模式、打破你的舒适区、接收新的内容。只要你学习的东西是正确的，突破之后，你就会感觉到自己的提高和进步。

    尤其达到瓶颈以后，针对缺陷的地方，刻意练习，反复练习，是突破瓶颈的最好方法。

    针对数据结构和算法的学习过程，就是要将学到的算法，自己去用代码多实现几遍，可以尝试不同的实现方式，多去思考，多去练习，达到真的理解掌握。尤其是有面试需求的，可能还需要去大量刷题才可以。

  • Feedback （反馈）

    学习最好的方法是就是多问，多思考，多交流。

    • 主动反馈

      去找比你做的更好的人，去GitHub，LeetCode 之类地方，看看高手是怎么实现的，学习思路总结自己。

    • 被动反馈

      有高手指点，但这个是可遇而不可求的。平时工作中可以做一些code review，大家一起相互学习、相互提高。

刷题

想要进入大公司，就要不断提高算法的编程水平，一个比较好的办法就是刷题，现在比较火的刷题网站是LeetCode，中文网址领扣(LeetCode)。刷题也不能盲目的刷，想要效率更高、效果更好，可以注意以下几点：

  • clarification

    明确题目的意思，认真读题，小心题目中的陷阱。

  • possible solutions

    • compare (time/space)
    • optimal

    做题时，不要满足于第一种解法。而是要想出尽量多的解法，通过分析时间、空间复杂度，选出其中最优解。

  • coding

    多写代码。

  • test cases

    做好测试用例，尽可能的全面，尤其是边界等易出错的地方。

在做一道题目之前，要考虑以上4点内容。完成题目之后，一定要多反馈，可以在论坛中看看别人的实现方式，交流一下自己的想法，吸取经验。

推荐书籍

最后

学习数据结构和算法的过程，是非常好的思维训练的过程，所以，千万不要被动地记忆，要多辩证地思考，多问为什么。遇见不懂的多看几遍，多交流。学习知识的过程就是反复迭代、不断沉淀的过程。坚持下去，coding的时候就会不由自主地考虑到很多性能方面的事情，时间复杂度、空间复杂度非常高的垃圾代码出现的次数就会越来越少。你的编程内功就真正得到了修炼。
