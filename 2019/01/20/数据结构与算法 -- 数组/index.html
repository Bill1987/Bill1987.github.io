数据结构与算法 -- 数组

发表于 2019-01-20 | 阅读次数:
字数统计: 1.4k | 阅读时长 ≈ 4

数组（Array）是一种线性表数据结构。它用一组连续的内存空间，来存储一组具有相同类型的数据，是最基础的数据结构。

线性表

线性表（Linear List）是最常用且最简单的一种数据结构，数据排成像一条线一样的结构。每个线性表上的数据最多只有前和后两个方向。常见的线性表结构有：数组，链表、队列、栈等。

非线性表

与线性表相对立的概念是非线性表，比如二叉树、堆、图等。之所以叫非线性，是因为，在非线性表中，数据之间并不是简单的前后关系。

随机访问

数组最大的两点就是：连续的存储区域 和 相同的数据类型。

正是因为这两个特点，数据就有了最大的特性，快速随机访问。

插入和删除

凡事有利就有弊，这种连续的存储结构虽然可以高效的随机访问，但是其插入和删除操作却十分低效。数组为了保证连续性，在做插入和删除操作时，就需要做大量的数据搬移工作。

假设要在”C”和”E”之间，插入”D”，就需要将EFG向后移一个地址，将D插入进去。同样，在删除的的时候，要将后面的数据，向前移动，以便让内存连续起来。

正因如此，数组的插入和删除是低效了，想象一下，如果在0的位置操作，就需要移动整个数组，因此它的时间复杂度就是$O(n)$。

为了解决低效的问题，在某些特殊情况下，可以改善一些效率。

在无序的数组中，插入新的数据，可以将新的数据赋值给要插入的位置元素，然后将原位置的的元素，追加到数组的队尾，这样会将时间复杂度降为$O(1)$，比如说，数组中有a，b，c，d，e，5 个元素。现在需要将元素 x 插入到第 3 个位置。我们只需要将 c 放入到 [5]，将 [2] 赋值为 x 即可。最后，数组中的元素如下： a，b，x，d，e，c。快排就是用的这个处理方式。但需要注意的是，一定是无序的数组才可以。同理在删除的时候，有可以这样操作，用尾元素覆盖要删除的元素，然后再清空尾数据。

在删除多个元素时，直接删除，会导致数据移动多次，效率低下。因此可以将要删的数据先标记出来，然后一起执行一次删除操作，这样数据仅需要移动一次，也会提高一些效率。

数组越界

在使用数组时，千万千万要小心越界的问题。数组在越界访问时，会产生很多莫名其妙的错误，在不同的环境下，表现出来的状况也不相同，所以不易查错。在有些语言或编译器下，可能会直接报错，这是非常理想的，但更多的是没有提示的，只有在运行时才能发现问题。

类似于这样的越界访问问题，很难调试，不易发现。有很多病毒就是利用这个原理访问非法地址，攻击系统。所以写代码的时候一定要警惕数组越界。

总结

数组是一个常用的基本结构，优缺点明显，如果你想要存放定量的数据，不经常修改，想随机访问，那数组是一个理想的结构。但要是经常对数据进行插入删除等操作，那就需要认真考虑一下了。
